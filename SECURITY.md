# 🛡️ Security Policy: MY-CREATIONS

## 1. Introduction
Welcome to the security policy for **MY-CREATIONS**. As we continue to ship new web tools and interactive experiments, maintaining a secure environment is a top priority. This document outlines how to report vulnerabilities and what to expect from the disclosure process.

## 2. Supported Versions
I actively provide security updates for the latest stable releases. Please ensure you are using the version marked as **Latest** on the repository's releases page.

| Version | Status | Notes |
| :--- | :---: | :--- |
| **v1.0.0** | ✅ Supported | Current Stable Release |
| **v0.9.x** | ⚠️ Limited | Beta Testing Phase |
| **< v0.9.0** | ❌ Unsupported | End of Life (EOL) |

## 3. Reporting a Vulnerability
If you discover a security flaw or a potential exploit, please do **not** open a public issue. Publicly disclosing a bug before a fix is ready can put other users at risk.

**Please follow these steps instead:**
1. Send a detailed email to **[sahoorojas@gmail.com]**.
2. Use the subject line: `Security Vulnerability: [Brief Description]`.
3. Include the specific URL or file path where the issue was found.
4. Provide a short description of the impact and how to reproduce the bug.

## 4. Disclosure Protocol
Once a report is received, the following process begins:
* **Acknowledgment:** I will respond within **48 hours** to confirm receipt.
* **Triage:** The vulnerability will be analyzed and assigned a priority level.
* **Remediation:** A fix will be developed and "shipped" to the `main` branch.
* **Release:** A new version will be published containing the security patch.

## 5. Scope
This policy covers all original source code, scripts, and CSS assets within the `sahoorojas-ship-it/MY-CREATIONS` repository. 

> [!IMPORTANT]
> This policy does **not** cover external third-party dependencies or browser-level security issues beyond the control of this project.

## 6. Recognition
I appreciate the efforts of security researchers and developers. Valid reports that lead to a security patch will be credited in the **Release Notes** of the next version.
